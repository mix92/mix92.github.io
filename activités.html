activité

C#

Cette formation m’a permis de découvrir le langage de programmation C# ainsi que son framework .NET et donc les points forts de cette formation étaient : Introduction au framework .NET et au langage C# : en bref, il s’agit d’un langage orienté objet, fortement typé, multi plateforme. Les bases de langage C# : cette partie présentait comment créer un premier programme C#, les éléments de syntaxe, les bases du typage, les différents data types à savoir type numérique, type Boolean, les chaines de caractères, les tableaux, les variables, les opérateurs. Les types C# : cette rubrique expliquait comment faire en C# les classes, Héritage, les types d’objets, les structures, les modificateurs, les interfaces. Pour aller un peu plus loin nous avons abordé la gestion des exceptions, les itérateurs, les attributs, les évènements et délégués. Bien que je ne sois pas doué en programmation, cette formation, je dirais que les 5 jours passés à Technofutur à étudier ce nouveau langage étaient très utile. En outre je dirais simplement merci aux formateurs pour leurs cours bien structurés et bien présentés.

java

Cette formation avait comme finalité l’initiation au langage JAVA, de sa syntaxe j’jusqu’au développement d’une petite application. Bien qu’ayant déjà eu les cours de java dans mon programme scolaire, elle m’a aussi permis d’élargir mon champ de connaissances par la découverte de nouvelles notions jamais abordées lors des à l’école. Durant cette formation, nous avons pu étudier plusieurs notions à savoir : Installation de outils et genèse d’un premier programme JAVA. Il existe plusieurs IDE permettant d’écrire un programme JAVA et, l’IDE utilisé ici était ECLIPSE au travers duquel nous avons pu écrire, compiler, et exécuter un premier programme JAVA. Au travers de ce premier programme, nous avons aborder quelques règles de programmation, notamment les conventions de nommage et syntaxique, utilisation des commentaires et l’amélioration de la lisibilité d’un programme. Variables, opérateurs et expression. Comme tous programmes, il existe une déclaration de variable et une variable peut avoir plusieurs types (entier, caractère, Boolean). Comme opérateurs, il existe les opérations de multiplication, addition, égalité, et soustraction. En ce qui concerne les expressions, on y trouve les combinaisons d’opérateurs. Les structures de contrôles. Cette partie développait les sélections alternatives (si, si-alors-sinon, sélection cas), les blocs d'instructions (notion de Début... Fin), les boucles itératives (tant que-répéter, répéter jusqu'à, pour-de- à), imbrication des instructions. Les procédures et les fonctions. A travers cette partie, nous avons développé la syntaxe de définitions : procédure, fonction Pourquoi sont-elles incontournables en programmation (réutilisabilité, lisibilité...), le passage de paramètres Le code retour d'une fonction, sensibilisation aux limites du passage de la valeur d'une variable, notion de passage par adresse, appel de fonctions.

sql admin

Cette formation vise à présenter l’ensemble des fonctionnalités offertes par le SGBD aux participants. Elle m’a permis de savoir comment manipuler SQL serveur afin d’effectuer les différentes tâches d’administration. Cette formation étant axé sur la pratique, avait comme contenu : Présentation de MSSQL server : à travers cette présentation, nous avons vu les éditions (entreprise, standard, développeur), l’architecture SQL server la typologie des fichiers (log et data base). Installation et configuration de SQL server : ici nous avons pu observer les versions de SQL server, le contrôle de l’installation, l’inscription du server et options de configuration. Outils et Interfaces d’administration : SQL Server Management Studio : explorateur d'objets et de solutions, analyseur de requêtes, SQL Computer Manager : services, configuration bibliothèques réseau, Outil SQLCMD : exécution de scripts SQL, dialogue interactif. Développement d’une base de données : cette partie expliquait les différents objets que pouvait avoir une data base. Il s’agit entre autres de : Création de tables, contraintes et relations Création et gestion des vues, Procédures stockées, Optimisation des requêtes (Partitionnement des tables et indexes, indexation des opérations en ligne et en parallèle, Maintenance et configuration des indexes) Backup et récupération : il s’agit ici d’une manière de Sauvegarder et restituer les données en cas de crash de server. Cela peut être fait de manière complète, différentielle, ou partielle. Grâce à cette formation, j’ai pu non seulement comprendre les bases du service DBA, mais aussi maitriser son outil.

ASP .NET MVC

Cette formation m’a permis de découvrir ASP .NET MVC. Il s’agit d’un Framework de développement web créé par Microsoft et permettant de réaliser des sites web dynamique grâce au modèle “MVC” (Model View Controller). Comme contunue de cette formation, nous avons pu aborder les éléments suivants : Introduction au développement web avec ASP .NET MVC : cette partie présentait la définition d’un site web, l’installation des outils de développement, et une brève introduction au langage HTML. Les bases d'ASP.NET MVC : cette partie expliquait le pattern MVC ASP.NET MVC en profondeur : cette partie présentait comment créer une route (consiste à aiguiller les requêtes des visiteurs dans notre application ASP. NET MVC), comment générer des tests automatiques (façon de vérifier qu’un système informatique fonctionne), et comment faire un model (correspond au M de MVC. Il s’agit des données et de l’état de l’application web, en général un ensemble de classes permettant d’accéder aux données) En définitive cette formation m’a permis d’élargira mon champ de connaissance en ce qui concerne le développement des sites web.

SSIS

SSIS (SQL server intégration service). Cette formation m’a permis de découvrir un outil SQL SERVER qui permet dans un projet décisionnel, de réaliser un ETL qui n’est rien d’autre qu’une extraction, transformation et chargement de données. Au travers de cette formation, nous avons abordé les notions suivantes : Fichiers sources : cette zone permet l’extraction des données en provenance de plusieurs types de fichiers à savoir : les fichiers plats (exemple : Excel, Word), les fichiers OLE DB qui représentent des fichiers sources des différents moteurs de bases de données. Zone de modification : cette zone est réservée aux différentes transformations que peut subir un fichier avant son chargement, cette zone permet de réaliser les conversions, permet une création de colonnes dérivées, de faire des unions, et des tris. Fichiers de destinations : cette zone permettra le chargement des données destination de plusieurs types de fichiers à savoir : les fichiers plats (exemple : Excel, Word), les fichiers OLE DB qui représentent des fichiers destinations des différents moteurs de bases de données. En définitive, je dirais que cette formation a été très utile pour moi surtout qu’elle m’a permis d’avancer rapidement sur une partie de mon TFE.
